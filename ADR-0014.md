# ADR-0014: DamageReport Numbering Convention

## Status
Approved

## Context
Official Palestinian damage reports require a unique, human-readable identifier that reflects their geographic and chronological context. Previous implementations used simple sequences or UUIDs, which are not suitable for official documentation.

## Decision
All Damage Reports will be assigned an official `ReportNumber` following this format:
`GovernorateCode-DirectorateCode-DamageYear-SequenceNumber`

### Components
1. **GovernorateCode**: 2-3 letter uppercase code (e.g., `NB` for Nablus).
2. **DirectorateCode**: 3 letter uppercase code (e.g., `NAB` for Nablus Directorate).
3. **DamageYear**: The year extracted from the `DamageDate` (not necessarily the creation year).
4. **SequenceNumber**: A 6-digit zero-padded number (e.g., `000001`).

### Rules
- The sequence is isolated per **Directorate** and **Damage Year**.
- The number is generated by the backend upon the first successful save of the report header.
- Once generated, the `ReportNumber` is immutable. It will not change if codes are updated in the reference tables.

## Consequences
- Requires a reliable sequence generation mechanism on the server to prevent duplicates under high concurrency.
- Mobile clients must wait for the first synchronization to display the official number.
- Enhances data integrity and traceability for official audits.
