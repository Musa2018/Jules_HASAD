## 3. Architecture

### Backend Structure (.NET 8 Clean Architecture)
- **Domain**: Pure entities (`Farmer`, `Farm`, `DamageReport`) and value objects. No dependencies.
- **Application**: CQRS pattern using **MediatR**. Contains Commands, Queries, and Validators (FluentValidation).
- **Infrastructure**: Implementation of `ApplicationDbContext` (EF Core), `IEmailService` (MailKit), and `IFileStorageService` (Local Disk).
- **API**: Controllers only. Uses API Versioning (v1).

### Mobile Structure (Flutter Feature-First)
- **Core**: Global services (Storage, Network, Router, L10n).
- **Features**: Sub-folders (e.g., `farmers`, `auth`) containing:
  - `data/`: Repositories and DTOs.
  - `domain/`: Business logic models.
  - `presentation/`: Riverpod providers and Widgets.

### Synchronization Architecture
1. **Local Write**: UI writes to Drift (SQLite) and adds an entry to `SyncQueue`.
2. **Background Engine**: `BackgroundSyncService` monitors connectivity.
3. **Queue Processing**: Items are sent to the API with their `ClientId`.
4. **Conflict Resolution**: "Server Wins" is the current implementation (server state overwrites local on 409 Conflict).

## 6. Technical Debt
1. **GoRouter `extra` Usage**:
   - **Reason**: Fast development of navigation.
   - **Impact**: State loss on Android process death.
   - **Priority**: High.
   - **Solution**: Refactor routes to use IDs in path parameters and fetch from local DB.
2. **Local Storage Cleanup**:
   - **Reason**: Attachments are deleted from DB but files remain on disk.
   - **Impact**: Storage bloat.
   - **Priority**: Medium.
   - **Solution**: Implement file-system cleanup in the Sync Service.
3. **Conflict UI**:
   - **Reason**: "Server Wins" policy is simple but leads to data loss.
   - **Impact**: User frustration.
   - **Priority**: High.
   - **Solution**: Implement a comparison screen for 409 errors.

## 13. Final Notes for the Next Engineer
- **Database**: The system expects a local SQL Server named `MUSA_PC`. If your instance is named differently, update `appsettings.Development.json`.
- **Identity**: The password policy is strict (12 chars, upper, lower, digit, symbol).
- **Sync**: The `BackgroundSyncService` uses a backoff of 0, 5, and 15 minutes. Check the mobile logs to monitor sync status.
- **Email**: Gmail SMTP requires an **App Password**. The current one in `appsettings` is temporary and should be rotated.
